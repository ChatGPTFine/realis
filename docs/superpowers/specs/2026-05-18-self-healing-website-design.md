# 自我疗愈网站 MVP 设计规格

## 概述

第一版要做一个“记录驱动的 AI 自我疗愈日记”。用户可以注册邮箱密码账号，记录今天发生的一件具体事情，由真实 AI 生成情绪觉察报告，再把报告保存到私密的时光画廊中。随着记录增加，系统逐步沉淀自己和重要他人的人际罗盘。

第一版是可部署上线的全栈 MVP，但严格收敛范围：不做支付、后台、公开社区、复杂 SEO、真实分享链接和完整 MBTI 测评。

## 目标用户

第一版服务的人群是：情绪内耗、关系敏感、想自我觉察的年轻用户。

典型特征：

- 容易反复回想关系或工作里的情绪事件。
- 想知道“这件事为什么这么刺痛我”。
- 需要温柔但有结构的方式来命名情绪、需求和模式。
- 更看重私密记录，而不是公开表达。

## 产品定位

一句话定位：写下今天发生的具体事情，AI 帮你看见情绪根源，并给出电影、书籍、音乐和行动组成的多维疗愈处方。

产品气质应该是安静、可信、私密、温柔但不空泛。它不是医疗服务，不是效率工具，也不是社交平台。

## 页面结构

### 首页

首页负责讲清楚产品价值，并引导用户开始第一次觉察。

主要入口：

- 开始觉察。
- 登录 / 注册。

首页需要让新用户快速理解：这里不是让你写一篇完整日记，而是从今天最具体的一件事开始，看见情绪背后的需求。

### AI 觉察

这是第一版的核心页面。

用户输入：

- 具体事件文本。
- 可选情绪标签。
- 可选相关人物或关系类型。
- 情绪强度。

AI 输出：

- 温柔回应：先接住用户当下的情绪。
- 情绪根源：指出真正刺痛的地方。
- 潜在需求：例如被看见、被尊重、边界、安全感、确定性。
- 认知或关系模式：例如过度自责、读心、灾难化、讨好、回避。
- 多维疗愈处方：电影、书籍、音乐、行动。
- 给未来自己的话：一句具体、温柔、有回看价值的话。
- 人际罗盘更新建议：这条记录对自己或相关人物画像提供了什么线索。

用户确认结果后，可以保存为一条私密记录。

### 时光画廊

时光画廊展示用户保存过的记录。第一版可以是时间轴或卡片流。

每条记录展示：

- 日期。
- AI 生成标题。
- 情绪标签。
- AI 摘要。
- 相关人物。
- 未来回看提示。

第一版不开放分享功能，但记录详情页可以预留未来“生成分享卡片”的位置。

### 人际罗盘

人际罗盘从保存记录中沉淀自己和重要他人的关系画像。

第一版支持：

- 自己。
- 父母。
- 朋友。
- 同事。
- 伴侣。
- 自定义重要人物。

每个人物画像展示：

- 关系类型和昵称。
- 相关记录数量。
- 常见触发点。
- 关系模式摘要。
- MBTI 倾向说明。
- 相处指南。

MBTI 内容必须被表达为“辅助理解的语言”，不能写成诊断、定性或固定人格标签。

### 登录 / 注册

第一版支持真实邮箱密码注册和登录。

未登录用户可以浏览首页，也可以先在 AI 觉察页输入草稿，但调用 AI 和保存记录前必须登录。登录后应尽量保留用户刚才输入的草稿，避免打断体验。

## 核心用户流程

1. 用户打开首页。
2. 用户点击“开始觉察”。
3. 用户写下一件具体事件，并补充情绪、人物和强度。
4. 如果用户未登录，系统要求登录或注册，并保留当前草稿。
5. 登录用户提交后，服务端调用 AI。
6. AI 返回结构化觉察报告。
7. 用户查看报告，并选择保存到时光画廊。
8. 保存后的记录进入画廊。
9. 保存记录同时为人际罗盘提供画像更新线索。
10. 用户未来可以按时间、情绪或人物回看记录。

第一版的体验闭环是：记录事件 -> AI 觉察 -> 保存 -> 回看 -> 关系沉淀。

## 数据模型

### User

由 Supabase Auth 管理。

字段：

- 邮箱。
- 密码哈希，由 Supabase 管理。
- 创建时间。

### ReflectionRecord

一条 AI 觉察记录。

字段：

- 用户 ID。
- 原始事件文本。
- 情绪标签。
- 情绪强度。
- 相关人物或关系类型。
- AI 生成标题。
- AI 摘要。
- 温柔回应。
- 情绪根源。
- 潜在需求。
- 认知或关系模式。
- 疗愈处方。
- 给未来自己的话。
- 未来回看时间或提示。
- 创建时间。
- 更新时间。

### PersonProfile

人际罗盘中的人物画像。

字段：

- 用户 ID。
- 关系类型。
- 昵称。
- 相关记录数量。
- 常见触发点。
- 关系模式摘要。
- MBTI 倾向说明。
- 相处指南。
- 创建时间。
- 更新时间。

### HealingPrescription

第一版中，疗愈处方可以作为结构化 JSON 存在 ReflectionRecord 中。

处方类型：

- 电影。
- 书籍。
- 音乐。
- 行动。

后续如果需要，可以再拆成独立推荐库。

## AI 行为设计

AI 语气采用混合型：

1. 先温柔接住用户情绪。
2. 再结构化分析事件、信念、需求和行动。
3. 最后用一句诗意但具体的话收束。

AI 应返回结构化 JSON，方便页面稳定渲染和数据库保存。

建议输出字段：

- `title`
- `summary`
- `gentle_response`
- `emotional_root`
- `underlying_needs`
- `pattern`
- `prescriptions`
- `future_self_note`
- `compass_updates`
- `safety_note`

## 安全与隐私

所有记录默认私密，只对用户本人可见。

要求：

- 数据按用户 ID 隔离。
- 服务端接口读取或写入私密数据前必须校验登录用户。
- OpenAI API Key 只能存在服务端，不能暴露给浏览器。
- 页面要明确说明：AI 觉察不是医学诊断，也不能替代心理治疗。
- 如果识别到自伤、自杀或急性危机表达，AI 应优先输出危机求助建议，鼓励用户立刻联系可信任的人、当地紧急服务或专业支持。

第一版不做公开分享、匿名社区、树洞广场、咨询师匹配。

## 技术架构

使用 Next.js 做全栈应用。

使用 Supabase 提供：

- 邮箱密码登录。
- Postgres 数据库。
- 行级权限，用于隔离不同用户的私密数据。

使用 OpenAI API 生成 AI 觉察报告。调用必须通过服务端接口完成。

使用 Vercel 部署 Next.js 网站，使用 Supabase 托管数据库和认证。

## 第一版范围

包含：

- 首页。
- AI 觉察页。
- 时光画廊页。
- 人际罗盘页。
- 邮箱密码登录和注册。
- 真实 AI 生成。
- 私密记录保存和查看。
- 基础人物画像更新。
- 可部署配置。

不包含：

- 支付。
- 后台管理。
- 公开社区。
- 真实分享链接。
- 复杂 SEO。
- 完整 MBTI 测评。
- 医学或心理治疗诊断。
- 咨询师匹配。

## 实施默认决策

- 使用 Next.js App Router。
- 调用 AI 前必须登录，避免成本失控和隐私边界模糊。
- 用户未登录时可以先写草稿，登录后继续提交。
- 人际罗盘第一版由 AI 输出的 `compass_updates` 更新，不额外做复杂总结任务。
- 视觉风格走安静、私密、年轻但不过度装饰的方向。
